## 🎯 什么是置信度评估？

**Confidence Check** 帮助AI学会诚实地评估自己的确定性，在回答问题时主动说明置信度。

> "知道你知道什么，也知道你不知道什么" —— 自我认知是可靠性的关键

---

## 📊 置信度等级

| 等级 | 标识 | 含义 | 行动 |
|-----|------|-----|------|
| 5 | ✅ | 确定 | 直接回答 |
| 4 | 👍 | 较确定 | 回答 + 注明小不确定点 |
| 3 | ⚠️ | 中等 | 回答 + 说明风险 |
| 2 | 🤔 | 不太确定 | 说明推测 + 建议核实 |
| 1 | ❓ | 猜测 | 明确表示不确定 |

---

## 🔍 评估四维度

```
         ┌──────────────────┐
         │   置信度评估      │
         └──────────────────┘
                   │
     ┌─────────────┼─────────────┐
     │             │             │
     ▼             ▼             ▼
 事实依据    逻辑一致性   上下文相关
     │
     ▼
   时效性
```

| 维度 | 问题 | 打分 |
|-----|------|------|
| **事实依据** | 有官方来源吗？多个来源验证？ | 1-5 |
| **逻辑一致性** | 内部逻辑自洽？推理合理？ | 1-5 |
| **上下文相关** | 回答相关？考虑约束？适合场景？ | 1-5 |
| **时效性** | 知识是否过时？领域变化快吗？ | 1-5 |

## 📝 使用示例

### 示例1：编程问题

**用户**：Python中list和tuple的区别？

**回答 + 置信度**：
```
## 回答

| 特性 | list | tuple |
|-----|------|-------|
| 可变性 | 可变 | 不可变 |
| 语法 | [1, 2] | (1, 2) |
| 性能 | 略慢 | 略快 |
| 用途 | 动态集合 | 固定记录 |

✅ **置信度：5/5** - 基于Python官方文档和常见知识
```

---

### 示例2：最新技术

**用户**：React 19有什么新特性？

**回答 + 置信度**：
```
## 回答

React 19 的主要新特性：

1. React Compiler (实验性)
2. 改进的 use() Hook
3. Actions 和表单处理改进

⚠️ **置信度：3/5** - 基于公开发布的信息，但可能不完整

**建议核实**：
- 官方 React 博客
- CHANGELOG.md
```

---

### 示例3：业务问题

**用户**：这个系统应该用什么架构？

**回答 + 置信度**：
```
## 回答

根据你的描述，有几种可能方案：

**方案A**：单体架构 → 适合小型、快速的MVP
**方案B**：微服务 → 适合大团队、需要扩展性
**方案C**：模块化单体 → 平衡选择

🤔 **置信度：2/5** - 这是基于一般原则的推测

**需要更多信息**：
- 团队规模？
- 预期用户量？
- 开发周期？
- 技术栈限制？
```

## ⚡ 快速评估检查表

```
回答前，先问自己：

1. 我从哪里知道这个？
   □ 官方文档
   □ 训练数据中的常见知识
   □ 合理推测

2. 有多确定？
   □ 确定正确
   □ 大概率正确
   □ 不太确定

3. 会误导用户吗？
   □ 不会
   □ 有小的不确定
   □ 有较大不确定
```

```markdown
| 上下文相关 | ⭐⭐⭐⭐⭐ | 适合场景 |

```

## 🚫 避免的错误

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| "这个肯定没问题" | "基于XX，我比较确定" |
| "绝对是这样" | "一般来说是这样，但有例外" |
| 假装很确定 | 诚实表达不确定性 |
| 不提供验证方式 | 建议核实渠道 |

---

## 💡 提升置信度技巧

如果评估发现置信度较低：

1. **补充条件**：添加"在...情况下"
2. **缩小范围**：针对特定场景回答
3. **提供备选**：给出多个可能选项
4. **明确假设**：说明基于哪些假设

---

## 🎯 什么时候触发

当用户问到：
- "你确定吗？"
- "这是官方的吗？"
- "这个信息准确吗？"
- 技术细节、版本差异、配置参数
- 最新发布、即将发布的功能
- 业务逻辑、设计决策
