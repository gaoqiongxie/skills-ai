## 功能定义

### 核心功能
帮助用户快速做出外卖选择，告别选择困难症。

### 输入参数
- 口味偏好（川菜/湘菜/粤菜/日料/韩餐/西餐/火锅/烧烤/面食/粥/轻食/甜品/等）
- 用餐人数
- 预算范围
- 特殊要求（不要辣/不要葱蒜/素食/低碳水/等）
- 心情状态（想吃辣的/想吃点清淡的/想补补/想吃甜的/等）

### 输出内容
1. **主推荐**：一个最终选择 + 推荐理由
2. **备选方案**：1-2个替代选项
3. **搭配建议**：主食+配菜+饮品推荐
4. **店铺参考**：适合的平台店铺类型

## 使用示例

### 示例1
**用户**: "两个人吃，预算100左右，想吃辣的重口味，不要太油腻"

**AI回复**:
```
🍜 今日推荐：川菜/湘菜

【主选】川菜
理由：重口味满足需求，辣度可选，两三个人吃分量刚好

推荐搭配：
├── 主菜：水煮肉片/酸菜鱼（选一）
├── 配菜：麻婆豆腐/干煸四季豆
├── 主食：米饭 ×2
└── 饮品：酸梅汤/凉茶（解辣）

预算参考：约80-100元

【备选】湘菜
- 小炒肉系列 + 剁椒鱼头
- 更辣更香
```

### 示例2
**用户**: "一个人，不想吃太油腻，想简单吃点"

**AI回复**:
```
🥗 今日推荐：轻食/面食

【主选】日式拉面
理由：清淡汤底，分量适中，配菜丰富

推荐搭配：
├── 主食：豚骨拉面/味噌拉面
├── 小食：煎饺/溏心蛋
└── 饮品：麦茶/乌龙茶

预算参考：约30-50元

【备选】港式云吞面
- 清淡鲜美
```

## 决策逻辑

### 口味 → 推荐菜系

| 口味 | 推荐菜系 |
|-----|---------|
| 想吃辣 | 川菜、湘菜 |
| 想吃清淡 | 粤菜、日料、港式 |
| 想吃重口 | 火锅、烧烤、麻辣烫 |
| 想吃简单 | 面食、盖饭、粥 |
| 想吃健康 | 轻食、沙拉、日料 |
| 想吃甜 | 甜品、日式料理 |
| 不知道 | 随机推荐 + 今日特惠 |

### 人数 → 分量建议

| 人数 | 建议 |
|-----|------|
| 1人 | 1主菜 + 1主食 或 1份套餐 |
| 2人 | 1-2主菜 + 2主食 + 小食 |
| 3-4人 | 2-3主菜 + 2主食 + 汤/小食 |
| 5人+ | 套餐/团购更划算 |

## 高级功能

### 今日特惠推荐
根据时间推荐：
- 工作日中午：商务套餐、盖饭、面食（出餐快）
- 工作日晚上：火锅、烧烤、日料（放松）
- 周末：网红店、约会餐厅、下午茶

### 节日/天气推荐
- 下雨天：火锅、麻辣烫、热汤面
- 夏天：凉面、冷面、寿司、沙拉
- 冬天：火锅、炖菜、热汤
- 节日：对应主题餐厅

## 附加价值

帮用户做决定后，可以：
1. 提醒营养搭配（建议加一份蔬菜/水果）
2. 提供凑单建议（差几元到满减）
3. 推荐类似店铺（货比三家）
