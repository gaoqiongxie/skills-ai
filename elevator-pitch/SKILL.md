## 功能定义

### 核心功能
帮你把复杂的产品/项目/自己，用1分钟/30秒/1句话讲清楚。

### 输入
- 介绍对象（产品/项目/自己/公司）
- 场景（面试/汇报/社交/路演/相亲）
- 时长要求（1句话/30秒/1分钟）
- 受众（专业人士/领导/投资人/普通用户）

### 输出
精炼的电梯演讲稿

---

## 电梯演讲公式

### 经典公式（SCQA）

```
S - Situation（情境）：在什么背景下...
C - Complication（冲突）：遇到了什么问题...
Q - Question（疑问）：用户/对方会问...
A - Answer（答案）：我们的解决方案是...
```

### 简化公式（3句话）

```
1. 我们是做什么的
2. 解决了什么问题
3. 给用户带来什么价值
```

### 人话公式（适合普通人）

```
我/我们 + 做/做了 + [具体事] + 帮助/让 + [用户/效果]
```

## 场景模板

### 1. 自我介绍（面试/社交）

**30秒版本**：
```
我叫[名字]，来自[城市/学校/公司]。

过去[X]年，我一直在做[领域]相关的工作，
主要经历是[核心经历1]和[核心经历2]。

最近在关注[当前工作/兴趣点]，
希望能够[目标/诉求]。

很高兴认识大家！
```

**1分钟版本**：
```
我叫[名字]，X[X]人，[X]年毕业于[学校/专业]。

毕业后一直从事[领域]工作，
之前在[公司A]做了[X]年，负责[工作内容]，
后来到[公司B]做了[X]年，做了[成就]。

[补充亮点，如：拿过什么奖、做出什么成绩]

目前在做[当前工作]，主要做[内容]，
希望能够在[方向]继续深耕。

很高兴认识大家！
```

### 2. 项目介绍

**30秒版本**：
```
这个项目是[项目名称]，
主要解决[用户痛点]的问题。

我们通过[核心方案]，
让用户可以[核心价值]。

目前已经[进展/成绩]，
下一步计划是[目标]。

[一句话总结亮点]
```

**1分钟版本**：
```
[项目名称]是我们为了解决[问题]而开发的产品。

背景是：[简单说行业痛点]

我们的方案是：[核心技术/方法]

核心功能包括：
1. [功能1]
2. [功能2]
3. [功能3]

上线后的数据：[关键指标]

和竞品比，我们的优势是[差异化亮点]。

希望[期望，如：获得资源支持/合作机会]
```

### 3. 产品介绍（对用户/客户）

**30秒版本**：
```
[产品名称]是[一句话定位]。

不管你是[目标用户]，
只要你有[用户场景]的需求，
就可以用我们的产品。

用了之后，你可以[核心价值]，
节省[具体收益]。

[一个用户证言或数据背书]
```

### 4. 工作汇报（对领导）

**30秒版本**：
```
领导，跟您汇报下[项目/工作]的进展。

目前[进展到什么阶段]，
[关键成果/数据]。

[遇到的问题，如有]
[需要的支持，如有]

下一步计划是[下一步]，
预计[时间/目标]。
```

## 经典示例

### 示例1：面试自我介绍

**输入**：我是Java开发，3年经验，想面试高级开发

**输出**：
```
面试官好，我叫XXX，3年Java开发经验。

之前在XX公司做了2年，
主要做[业务领域]的后端开发，
用Java + Spring全家桶，
主导过XX系统的重构。

后来在YY公司做了1年，
带队完成了XX项目，
QPS从XXX提升到XXX。

最近在看高级开发的岗位机会，
希望能在技术深度和架构能力上有更多成长。

请多指教！
```

### 示例2：项目路演

**输入**：做一个面向Z世代的社交App

**输出**：
```
我们做了一个面向Z世代的新型社交产品——[名字]。

现在的年轻人，
觉得微信太无聊、抖音太被动，
找不到一个能真实表达自己、找到同好的地方。

我们的解决方案是：
用「兴趣标签」+「即时匹配」+「匿名互动」，
让用户可以：
- 不用露脸也能交朋友
- 找到真正志同道合的人
- 表达真实的自己

上线3个月，
DAU 10万+，次日留存45%，
用户平均时长25分钟。

我们正在寻求A轮融资，
计划用这笔钱快速迭代和获取用户。

希望各位投资人多多支持！
```

### 示例3：工作汇报

**输入**：向领导汇报618大促活动结果

**输出**：
```
领导，跟您汇报下618大促的结果。

整体来看，GMV超预期完成，
达到了[X万]，同比增长[X%]。

其中[A品类]表现最好，
销量[X]，占比[X%]。

活动期间我们做了[核心策略]，
配合[营销动作]，
整体的转化率从[X%]提升到[X%]。

用户反馈方面，[正面反馈描述]。

整体来说这次活动还是比较成功的，
复盘会我们总结了几个可以优化的点，
下次大促可以做得更好。
```

## 注意事项

### ✅ 应该做的
1. **提前准备**：重要场合提前写好、练熟
2. **目标明确**：想达到什么目的，就重点说什么
3. **数据说话**：有数据就上数据
4. **看人说话**：技术说技术细节，领导说价值和结果
5. **控制时长**：宁可短一点，不要超时

### ❌ 不应该做的
1. 一上来就讲技术细节
2. 把所有功能都讲一遍
3. 背简历式的流水账
4. 说太多专业术语
5. 超时！（最重要的一点）

---

## 练习技巧

1. **对着镜子练**：看表情和肢体语言
2. **录音回听**：听起来顺不顺
3. **计时**：确保不超时
4. **找人试讲**：让朋友/同事给你反馈
5. **准备QA**：预测可能被问的问题
