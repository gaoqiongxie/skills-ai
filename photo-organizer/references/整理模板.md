## 六、整理规范建议

### 命名规范

**文件夹命名**：
```
✅ 正确格式：
2024-01-15_宝宝百天
2024-05-01_东京Day1
2024-06-18_老公生日

❌ 避免格式：
新建文件夹 (1)
IMG_1234
无标题
```

**照片命名**：
```
✅ 推荐：
宝宝_2024-01-15_百天照_001.jpg
东京_2024-01-01_浅草寺_001.jpg

❌ 避免：
IMG_1234.JPG
微信图片_20240115.jpg
```

### 定期整理频率

| 类型 | 建议频率 |
|-----|---------|
| 手机照片 | 每月整理一次 |
| 旅行照片 | 旅行结束后一周内 |
| 宝宝照片 | 每周或每月 |
| 工作素材 | 用完即整理 |

### 清理规则

**必删**：
- 完全重复的照片（保留一张）
- 模糊不清的废片
- 试拍照片
- 截图类（单独归类）

**可删**：
- 相似照片保留最佳
- HDR/连拍保留最好的一张
- 修图后的原图（如果修图版更好）

**必留**：
- 重要证件照电子版
- 有意义的合照
- 重要事件记录
- 宝宝成长里程碑

## 七、iPhone/安卓整理技巧

### iPhone原生功能
- 相簿 → 创建相簿
- 回忆 → 自动生成视频
- 共享相簿 → 家人共享

### 推荐第三方工具
- Google Photos（智能分类）
- Timehop（往年今日）
- 腾讯相册管家（微信照片整理）
- 存储空间清理类App

### 云备份建议
- iCloud / Google Photos / 腾讯相册
- 重要照片双重备份
- 定期导出到电脑

## 八、不同需求的整理方案

### 方案A：极简主义
只保留最精华的100张照片，其他全部删除或归档。

### 方案B：时间线法
按年份→月份→事件的严格时间线整理。

### 方案C：人物中心法
以人物为主角，所有照片按"宝宝"、"老公"、"自己"分类。

### 方案D：事件驱动法
以事件为线索，"婚礼"、"旅行"、"宝宝周岁"等。

### 方案E：混合法（推荐）
时间线+人物+事件三者结合，最实用！

```
📂 2024/
├── 📂 01-二月 春节/
│   ├── 📂 全家福
│   ├── 📂 宝宝新年装
│   └── 📂 烟花
├── 📂 05-五月 马尔代夫/
│   ├── 📂 Day1
│   ├── 📂 Day2
│   └── 📂 宝宝沙滩照
└── ...
```

*整理照片的本质是：让美好的回忆更容易被找到和回顾*
