## 功能定义

### 核心功能
根据你描述的工作内容，自动生成简洁、专业、有亮点的周报。

### 输入
- 本周做了哪些工作
- 下周计划做什么
- 遇到的问题/需要协助（可选）
- 项目背景（可选）

### 输出
结构化的周报文档

## 周报类型

### 1. 个人周报（员工用）

**特点**：
- 突出个人贡献
- 数据化呈现
- 展示解决问题的能力

**常用句式**：
- "完成了XXX"
- "优化了XXX，效率提升X%"
- "解决了XXX问题"
- "协助XXX完成了XXX"

### 2. 团队周报（leader用）

**特点**：
- 突出团队整体成果
- 各成员工作汇总
- 团队目标和进度的把控

### 3. 项目周报

**特点**：
- 项目进度百分比
- 里程碑完成情况
- 风险和问题清单

## 工作描述 → 周报句式

### 描述完成的工作

| 你说的 | 周报写法 |
|-------|---------|
| 做了个页面 | 完成了XX功能页面的设计和开发 |
| 修了个bug | 修复了XX模块的XX问题，提升了系统稳定性 |
| 开会讨论了 | 参与了XX评审会，明确了XX方向 |
| 写了文档 | 编写了XX文档，完善了XX规范 |
| 做了测试 | 完成了XX功能的测试，测试覆盖率XX% |
| 和别人对了需求 | 与XX团队进行需求对齐，确认了XX方案 |

### 量化工作成果

| 场景 | 量化方式 |
|-----|---------|
| 开发 | 完成X个功能模块，代码量X行 |
| 测试 | 执行X个用例，发现X个bug |
| 运营 | 活动参与人数X，转化率X% |
| 销售 | 完成业绩X万，环比增长X% |
| 客服 | 处理工单X个，平均响应时长X分钟 |

## 周报写作技巧

### ✅ 应该做的
1. **数据说话**：用数字量化成果
2. **突出亮点**：挑最重要的说
3. **展现思考**：不只是列任务，要写成果
4. **提前规划**：下周计划要具体可执行
5. **及时反馈**：有问题主动提出

### ❌ 不应该做的
1. 流水账式记录（"上午写了代码，下午开了会"）
2. 只有任务没有成果
3. 夸大或虚构工作
4. 报喜不报忧（有问题要提）

|-----|---------|

| 周报 | 每周五下班前 或 周一上班前 |
| 月报 | 每月最后一天 或 每月第一天 |
