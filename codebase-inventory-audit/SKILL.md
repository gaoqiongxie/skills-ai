# 🗂️ Codebase Inventory Audit — 代码库清单审计

自动盘点代码库，识别技术债务和优化机会。

---

## 🎯 核心能力

| 能力 | 说明 |
|------|------|
| **结构可视化** | 生成模块依赖图、调用关系图 |
| **孤儿代码识别** | 找出无引用的函数、类、文件 |
| **重复逻辑检测** | 识别代码重复、相似函数 |
| **文档缺口扫描** | 找出缺少注释/文档的公共 API |
| **技术债务量化** | 给代码库健康度打分 |
| **依赖分析** | 第三方库使用情况、过时依赖 |

---

## 📊 审计维度

### 1. 代码结构健康度

```
项目: my-project
├── 模块数量: 23
├── 文件数量: 487
├── 代码行数: 42,380
├── 平均文件行数: 87
├── 最大文件: src/services/legacy/monolith.ts (2,340 行) ⚠️
└── 循环依赖: 3 处 ❌
```

### 2. 孤儿代码扫描

| 类型 | 数量 | 示例 |
|------|------|------|
| 未引用函数 | 12 | `utils/legacyHelper.ts:calculateOldTax` |
| 未使用类 | 3 | `models/DeprecatedUser.ts` |
| 死代码文件 | 5 | `hooks/useOldApi.ts` |
| 未使用导出 | 18 | `constants/oldConfig.ts` |

### 3. 重复代码检测

| 相似度 | 位置 A | 位置 B | 建议 |
|--------|--------|--------|------|
| 85% | `auth/login.ts:45-67` | `auth/register.ts:32-54` | 提取公共函数 |
| 72% | `orders/validate.ts` | `payments/validate.ts` | 创建共享校验器 |

### 4. 文档缺口

| 文件 | 公共 API | 有文档 | 缺口 |
|------|---------|--------|------|
| `api/user.ts` | 8 | 3 | 5 |
| `services/order.ts` | 12 | 2 | 10 |
| `utils/date.ts` | 6 | 6 | 0 ✅ |

## 🚀 使用方式

### 方式一：全量审计

```
对当前项目执行代码库审计
```

### 方式二：定向扫描

```
扫描 src/services 目录的孤儿代码
```

### 方式三：债务清理计划

```
根据审计结果生成技术债务清理计划
按 ROI 排序
```
