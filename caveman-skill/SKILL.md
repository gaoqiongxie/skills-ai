# 🪨 Caveman Skill — Token 优化器

用"洞穴人语"让 AI 输出更精简，token 成本大幅降低。

---

## 🎯 核心能力

| 能力 | 效果 | 场景 |
|------|------|------|
| **输出压缩** | 减少 ~75% 输出 token | 日常对话、代码解释、文档生成 |
| **输入压缩** | 记忆文件减少 ~46% | 长上下文加载、CLAUDE.md 压缩 |
| **提交精简** | 单条提交信息 | Git 提交规范化 |
| **PR 速评** | 一行代码审查 | 快速代码评审 |
| **统计追踪** | 终身节省 token 统计 | 成本量化 |

---

## 🚀 使用方式

### 方式一：全局启用

```
启用 caveman 模式
```

AI 会以精简风格回复所有后续对话。

### 方式二：临时使用

```
/caveman 解释一下这段代码
/caveman 总结这个 PRD
/caveman 压缩这个文件
```

### 方式三：输入压缩

```
/caveman:compress <文件内容>
```

将长文本压缩为洞穴人语摘要，用于记忆文件或上下文加载。

---

## 📝 输出风格对比

### 普通模式（高 token）

```
这段代码实现了一个用户认证模块，它使用了 JWT Token 进行身份验证。
当用户提交用户名和密码时，系统首先会验证这些凭据的有效性，
然后生成一个包含用户ID和过期时间的 Token 返回给客户端。
```

### Caveman 模式（低 token）

```
用户认证。JWT Token。验证凭据。生成Token。返回客户端。
```

**节省：~75% token**

## 💡 最佳实践

### DO
- ✅ 在需要快速浏览大量信息时启用
- ✅ 用于长文档的初步概览
- ✅ 压缩 CLAUDE.md 等记忆文件以节省上下文窗口
- ✅ 日常开发对话中降低 API 成本

### DON'T
- ❌ 初次学习复杂概念时（需要详细解释）
- ❌ 需要精确法律/医疗建议时
- ❌ 与团队分享最终文档时（需要完整表达）
