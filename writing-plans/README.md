## 🎯 什么是实施计划？

**Writing Plans** 将复杂任务分解为清晰的、可执行的步骤，确保每次只专注于一件事。

> "计划是成功路线图" —— 没有计划的行动是盲目的

---

## 📐 计划结构

```markdown
## 任务名称

### 概述
[简要描述]

### 阶段1: [名称]
**目标**: [完成什么]
**步骤**:
1. 步骤1
2. 步骤2
**验证**: [ ] 通过

### 阶段2: [名称]
...

### 风险与应对
| 风险 | 措施 |
|-----|------|
| 风险1 | 措施1 |

### 完成标准
- [ ] 标准1
- [ ] 标准2
```

---

## 🔄 阶段划分策略

### 按功能模块
```
阶段1: 数据层 (Entity, Repository)
阶段2: 业务层 (Service)
阶段3: 接口层 (Controller)
阶段4: 前端
阶段5: 测试
```

### 按风险高低
```
阶段1: 低风险（基础）
阶段2: 核心（重点测试）
阶段3: 高风险（仔细验证）
阶段4: 完善
```

### 按价值交付
```
阶段1: MVP（最小可用）
阶段2: 完整功能
阶段3: 优化完善
```

---

## 📝 分解原则

| 原则 | 说明 | 示例 |
|-----|------|------|
| **原子化** | 单一职责 | "添加用户实体"而非"做用户模块" |
| **可验证** | 明确标准 | "接口返回正确数据" |
| **时间合理** | 5-30分钟 | 太长则拆分 |

## 📋 模板库

### 简单任务 (1-2小时)

```markdown
## 任务：添加搜索功能

### 步骤
1. 添加搜索接口
2. 实现搜索逻辑
3. 添加前端搜索框
4. 联调测试

### 验证
- [ ] 搜索结果正确
- [ ] 性能 < 200ms
```

### 中等任务 (半天-1天)

```markdown
## 任务：用户评论功能

### 阶段1: 数据层
- 创建评论表
- 添加索引

### 阶段2: 后端API
- 评论列表
- 发布评论
- 删除评论

### 阶段3: 前端
- 列表组件
- 输入框

### 阶段4: 测试
- 联调测试
- 边界测试
```

### 大型任务 (3天+)

```markdown
## 任务：支付系统

### 阶段1: 基础架构 (Day 1)
**目标**: 建立框架

### 阶段2: 微信支付 (Day 2)
**目标**: 第一个插件

### 阶段3: 支付宝 (Day 3)
**目标**: 第二个插件

### 阶段4: 测试完善 (Day 4)
**目标**: 系统稳定
```

## ⚡ 执行技巧

### 番茄工作法
```
🍅 25min: 步骤1-2
☕ 5min: 休息
🍅 25min: 步骤3-4
☕ 5min: 休息
🍅 25min: 验证
```

### 检查点
每个阶段后：
1. 提交代码
2. 运行测试
3. 确认无误
4. 下一阶段

### 调整计划
发现不合理时：
- ❌ 硬着头皮做
- ✅ 停下来重新分析
- ✅ 更新计划
- ✅ 继续执行

---

## ⚠️ 注意事项

| ✅ 要 | ❌ 不要 |
|------|--------|
| 步骤具体可执行 | 过度计划 |
| 明确验证点 | 模糊步骤 |
| 识别风险 | 没有余量 |
| 番茄工作法 | YAGNI |
